# 끄적끄적 고도화 기획

## 제품 방향

끄적끄적은 친구나 연인이 매일 같은 질문을 보고, 글보다 가벼운 낙서로 서로의 하루와 마음을 확인하는 1:1 그림 문답 서비스다. 핵심은 많이 말하지 않아도 서로를 조금 더 알게 되는 경험이다.

## 핵심 기능

- 프로필 선택: 한 기기에서 여러 프로필을 만들고, 프로필마다 다른 사람과 연결할 수 있다.
- 둘만의 방: 초대 코드로 1:1 방을 만들고, 방이 끊겨도 기존 기록은 유지한다.
- 오늘의 질문: 공통, 친구, 연인 문구를 같은 날짜 안에서 다르게 제공한다.
- 낙서 제출: 하루 한 번 제출하고, 당일에는 한 번 수정할 수 있다.
- 동시 공개: 둘 다 제출하면 서로의 그림이 열린다.
- 그림일기: 날짜별로 그림과 질문을 모아 본다.
- Google 저장: 전체 로그 시트와 방별 시트에 이벤트, 그림, 채팅 데이터를 남긴다.

## 설정으로 모을 서브 기능

- 관계 유형: 가까운 사람, 친구, 연인
- 하루 기준 시간대: 기본 한국시간, 사용자가 변경 가능
- 다크모드
- 닉네임 수정
- 방별 알림 톤
- 데이터 내보내기
- 방 나가기와 기록 보존 안내

## 데이터 구조 원칙

- `전체 로그` 시트는 모든 이벤트를 시간순으로 저장한다.
- 새 방이 만들어지면 `방_{id}` 시트가 자동 생성된다.
- 각 시트의 첫 행은 컬럼 설명을 담는다.
- 방이 끊겨도 시트와 Drive 파일은 삭제하지 않는다.
- Drive에는 원본 PNG를 저장하고, Sheets에는 파일 ID와 URL을 함께 기록한다.
- 서버 상태는 Google Sheets 이벤트 로그에서 복원 가능해야 한다.

## 10가지 페르소나 피드백

1. 장거리 연애 중인 29세 직장인: 긴 통화보다 하루 한 장이 부담이 적다. 시간대 설정과 공개 알림이 중요하다.
2. 매일 연락은 어려운 31세 연인 사용자: 질문이 너무 달달하기만 하면 피곤하다. 현실적인 감정 질문이 좋다.
3. 친한 친구와 쓰는 24세 대학생: 연인 워딩이 보이면 어색하다. 친구용 질문과 웃긴 이모티콘이 필요하다.
4. 감정을 말로 표현하기 어려운 27세 사용자: 채팅보다 낙서가 먼저라 편하다. 제출 전 임시저장이 중요하다.
5. 기록을 좋아하는 34세 사용자: 월별 달력과 내보내기가 있으면 오래 쓴다.
6. 바쁜 26세 사용자: 하루 질문이 길면 이탈한다. 첫 화면에서 바로 오늘 방으로 들어가야 한다.
7. 사귄 지 얼마 안 된 22세 연인 사용자: 너무 깊은 질문은 부담스럽다. 관계 초반용 질문 난이도 조절이 필요하다.
8. 오래 만난 36세 연인 사용자: 익숙함을 깨는 질문이 좋다. 과거, 미래, 후회, 꿈을 다루는 질문이 필요하다.
9. 여러 친구와 쓰는 25세 사용자: 방이 여러 개일 때 프로필과 방 전환이 쉬워야 한다.
10. 개인정보에 민감한 30세 사용자: Google Drive/Sheets 저장 위치와 삭제 정책이 명확해야 안심한다.

## 추가하면 좋은 기능

- 오늘의 한 줄 댓글: 그림 아래에 짧은 말을 남기고 수정할 수 있다.
- 월별 방명록: 달력 아래에 그달의 감정이나 사건을 남긴다.
- 반응 버튼: 표정 3개와 하트, 별 중 하나를 눌러 가볍게 반응한다.
- 질문 난이도: 가벼움, 깊음, 회상, 미래 중 선택한다.
- 둘만의 타임캡슐: 특정 날짜에 함께 열 편지나 그림을 예약한다.
- 기념일 질문: 방을 만든 날, 30일, 100일에 특별 질문을 준다.
- 꿈꾸는 지도: 같이 가고 싶은 곳, 살고 싶은 공간, 해보고 싶은 일을 그림으로 모은다.
- 추억 리플레이: 지난 그림을 랜덤으로 꺼내 “그때의 우리”를 다시 본다.
- 데이터 백업 확인: 마지막 Google 저장 성공 시간을 설정 화면에 표시한다.

## 우선순위

1. Google Sheets/Drive 저장 복원과 방별 시트 자동 생성
2. 프로필 선택, 닉네임 수정, 관계 유형 설정
3. 그리기 도구 확장
4. 채팅과 이모티콘 반응
5. 시간대별 하루 기준 적용
6. 방 나가기와 기록 보존 정책
7. 타임캡슐, 기념일 질문, 추억 리플레이
